# Release v1.2.0

Six workflow skills for `/lexis` — clarify, slice, debug, ground, compare, and scenario-walk before you code.

## Install / upgrade

```bash
npm install -D @draig/lexis-two@1.2.0
# or refresh slash commands (OpenCode global):
npx @draig/lexis-two install --host opencode --scope global --yes
```

Restart OpenCode after upgrading so new slash commands appear in autocomplete.

## New commands

| Command | Skill folder | Purpose |
| ------- | ------------ | ------- |
| `/lexis doubt` | `skills/lexis-two-doubt-driven/` | Clarify ambiguous requirements (max 3 questions, no code) |
| `/lexis incremental` (`inc`) | `skills/lexis-two-incremental-impl/` | Smallest vertical slice first |
| `/lexis debug` (`triage`) | `skills/lexis-two-debug-triage/` | Minimal repro and fix |
| `/lexis source` (`src`) | `skills/lexis-two-source-driven/` | Repo- and docs-grounded design |
| `/lexis predict` | `skills/lexis-two-predict/` | Proposed vs lazy comparison |
| `/lexis scenario` | `skills/lexis-two-scenario/` | Happy / edge / failure scenarios |

Legacy `/lexis-two-*` names still work with a deprecation notice. Canonical user-facing form is `/lexis <subcommand>`.
